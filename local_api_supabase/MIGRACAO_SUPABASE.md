# Migracao para Supabase

Este roteiro foi preparado com base no schema atual do MySQL local `reserva_escolar_v2`.

## Base atual mapeada

- `schools`: 3
- `users`: 7
- `resources`: 34
- `bookings`: 11
- `booking_lessons`: 25
- `class_groups`: 29
- `subjects`: 18
- `lesson_slots`: 23
- `resource_categories`: 3

Validacoes feitas na base atual:

- `orphan_completed_by = 0`
- `orphan_cancelled_by = 0`
- `duplicate_school_code = 0`
- `duplicate_api_token_nonnull = 0`

## Pontos de compatibilidade com PostgreSQL

- `AUTO_INCREMENT` foi convertido para `generated by default as identity`.
- `enum(...)` foi convertido para `varchar` com `check`, o que evita friccao em import e continua compatível com a API.
- `tinyint(1)` foi mantido como `smallint` com `check (active in (0, 1))`.
  Isso preserva o comportamento atual do PHP, que trata `active` como numero.
- `timestamp` e `datetime` foram mapeados para `timestamp`.
- Foi adicionada a FK de `completed_by_user_id`, que nao existia no dump MySQL, mas os dados atuais estao consistentes para isso.

## Ordem recomendada

1. Criar um projeto no Supabase.
2. Abrir o SQL Editor.
3. Rodar [schema_postgres.sql](/opt/lampp/htdocs/reserva_escolar_api_v2/supabase/schema_postgres.sql).
4. Exportar os dados do MySQL.
5. Importar os CSVs na seguinte ordem:

- `schools`
- `resource_categories`
- `users`
- `class_groups`
- `subjects`
- `lesson_slots`
- `resources`
- `bookings`
- `booking_lessons`

6. Rodar [validation_queries.sql](/opt/lampp/htdocs/reserva_escolar_api_v2/supabase/validation_queries.sql) no Supabase.
7. Configurar a API com `RESERVA_DB_URL` apontando para o pooler do Supabase.
8. Fazer smoke test pelos endpoints principais.

## Exportacao da base atual

Use:

```bash
cd /opt/lampp/htdocs/reserva_escolar_api_v2
chmod +x supabase/export_mysql_to_csv.sh
./supabase/export_mysql_to_csv.sh
```

Isso gera CSVs em `supabase/export_csv`.

## Ajustes manuais esperados na importacao

- Se algum campo de texto contiver virgula ou quebra de linha, o script CSV simples pode exigir ajuste. Como sua base atual parece pequena, vale conferir os arquivos antes do import.
- Se preferir mais robustez, exporte em TSV e converta com quoting apropriado antes do upload no Supabase.
- Depois da importacao, confira se `api_token` e `api_token_expires_at` vieram corretamente, pois impactam login persistente.

## Smoke test minimo

Depois de apontar a API para o Supabase:

- `POST /login.php`
- `GET /get_resources.php?school_id=...`
- `GET /get_my_bookings.php?school_id=...&user_id=...`
- `GET /get_all_bookings.php?school_id=...`
- `POST /create_booking.php`
- `POST /cancel_booking.php`
- `POST /complete_booking.php`

## Risco residual

O schema base e a camada de conexao ja estao preparadas, mas ainda pode haver consultas SQL menores nao exercitadas no dia a dia que precisem de ajuste fino quando a API rodar 100% em PostgreSQL.
